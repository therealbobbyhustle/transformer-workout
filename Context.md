PROJECT CONTEXT DOC
Transformer Workout | V1 Challenge Dashboard and Core Functionality

PROJECT OVERVIEW

We are building the V1 of a fitness challenge platform for Transformer Workout.

This is not a hardcore fitness app for bodybuilders or fitness influencers.

This platform is for busy working adults, especially moms and dads in their 30s and 40s, who have been neglecting their health because of work, stress, and family responsibilities.

The emotional message behind the brand is:

"If Your Body Crashes, Your Whole House Crashes."

The product should feel simple, supportive, structured, and easy to follow.

The user should feel like they can log in, see exactly what to do, complete it, and feel accomplished.

This is not a complex fitness tracker.
This is a guided completion-based challenge experience.

CORE PRODUCT

Working product:
A self-paced 21-day fitness and nutrition challenge

Working title:
Strong for the House: 21-Day Challenge

This may change later, so keep naming editable.

The challenge is self-paced, but visually organized in a recommended Monday through Sunday structure.

Users can complete the days at their own convenience.
They are not locked to real calendar dates.
They should not feel punished if they miss a day.

The product should help users:
1. follow a simple workout plan
2. get basic nutrition guidance
3. track completion with green check marks
4. feel progress visually
5. finish the challenge and receive a congratulations message
6. see an upsell into the next membership/program at the end

TARGET USER

Target audience:
- busy moms and dads
- average working Americans
- adults in their 30s and 40s
- people trying to get back on track
- not fitness influencers
- not advanced athletes
- people who want structure without complexity
- people who need motivation and clarity, not data overload

USER EXPERIENCE GOAL

The dashboard should feel like this:

- log in
- see today’s workout
- press play
- follow the video
- mark complete
- see a green check mark
- feel good
- move to the next day
- finish the challenge
- see congratulations + next step offer

The experience should feel easy and intuitive.

The platform should answer one main question:
"What do I need to do today?"

It should not feel like a complicated content portal.

V1 FEATURE PHILOSOPHY

Keep V1 simple.
Do not overbuild.
The app should focus on completion, progress, and clarity.

This is more of a guided challenge dashboard than a full fitness app.

Avoid:
- calorie counting systems
- macro tracking
- barcode scanners
- advanced analytics
- social feed complexity
- wearables
- leaderboards
- heavy gamification
- complicated onboarding
- too many menu items
- too many decisions per screen

V1 CORE SECTIONS

The main user dashboard should include these sections:

1. Dashboard Home
2. Workouts / Challenge Days
3. Nutrition Guidance
4. Weekly Check-In / Progress
5. Completion / Congratulations / Upsell

DASHBOARD HOME REQUIREMENTS

The dashboard home is the main screen after login.

This should be the most important page in the product.

The user should instantly see:

1. Welcome message
2. Current progress
3. Their next recommended day
4. Today’s workout or next step
5. Simple habit or nutrition reminder
6. Weekly challenge layout
7. Completion status

Suggested Dashboard Home elements:

A. Welcome Header
- "Welcome back, [First Name]"
- Day X of 21
- short motivational/supportive line

Examples:
- Keep stacking wins
- You’re building momentum
- One day at a time
- Strong households start with strong habits

B. Progress Summary
Show:
- days completed out of 21
- completion percentage
- current streak if included
- next milestone if included

Example:
- 6 of 21 days complete
- 29% complete
- 3-day streak

C. Main Action Card
This is the most important card.
It should show the user’s next recommended action.

Example:
Today’s Workout
Full Body Strength A
Time: 22 minutes
Focus: Strength + Energy
Button: Start Workout

Or if the next item is a recovery day:
Today’s Recovery Session
Walk + Core
Time: 15 minutes
Button: Start Day

D. Completion CTA / Mark Complete Button
After workout content is viewed/completed, allow user to mark the day complete.

Example button:
- Mark Day Complete
- Complete Today’s Workout

E. Completion Feedback
Once completed, show:
- green check mark
- short positive message
- automatic progress update
- next day preview

Example:
Day 6 complete
Great job. You showed up for yourself today.

F. Weekly Challenge Layout Preview
Show the weekly structure organized visually Monday through Sunday.

Each day should have a status:
- completed = green check
- current next day = highlighted
- future day = inactive or lighter
- optional locked state if needed

WORKOUT STRUCTURE CONTEXT

The challenge is 21 days long.

It is self-paced but visually structured in 3 weeks:

Week 1: Reset
Week 2: Build
Week 3: Lock In

Each week follows a Monday through Sunday rhythm.

Recommended schedule:

Monday: Workout
Tuesday: Recovery / Core / Walk
Wednesday: Workout
Thursday: Mobility / Recovery
Friday: Workout
Saturday: Workout
Sunday: Rest + Check-In

The user can complete these on their own timeline.

The program includes 4 workout days per week.

WORKOUT PAGE REQUIREMENTS

Each workout day should have its own page or view.

A workout page should include:

1. Day title
2. Week title
3. Estimated duration
4. Focus or workout type
5. Embedded video area or video player placeholder
6. Optional short description or coaching note
7. Optional beginner / stronger option labels
8. Mark Complete button
9. Link/button to next day after completion

Suggested fields:
- dayNumber
- weekNumber
- weekLabel
- dayLabel
- title
- duration
- category
- description
- videoUrl or placeholder
- completed
- completedAt

The workout page should feel dead simple.
The user should not have to dig for what to do.

PROGRESS TRACKER REQUIREMENTS

The progress tracker is one of the key value drivers.

It should be simple and visual.

Progress should focus on completion, not advanced body metrics.

Core tracking should include:
- days completed
- percent complete
- current day / next day
- green checks for completed days

Optional:
- streak count
- weekly completion summary

The challenge map should show all 21 days in 3 week sections.

Visual states:
- completed
- current / next recommended
- upcoming
- optional missed/incomplete

If a user misses days, do not punish them or shame them.
Use supportive language like:
- Welcome back
- Let’s keep going
- You’re not behind, just continue your next step

NUTRITION SECTION REQUIREMENTS

The platform should include simple nutrition guidance so the challenge feels like a full action plan.

This is NOT a calorie-counting app.

Do not build:
- macro calculator
- food log
- calorie database
- barcode scanner

Instead, build a lightweight nutrition guidance section.

Nutrition section should include:

1. Simple Nutrition Game Plan
A short explanation of how to eat during the challenge.

2. Daily Nutrition Wins
Simple checklist-style guidance such as:
- Drank my water
- Ate protein with meals
- Made better food choices
- Stayed mostly on plan

3. Meal Suggestions
Simple, realistic meal ideas for busy adults.
Examples:
- eggs + fruit
- Greek yogurt + berries
- grilled chicken salad
- turkey wrap + fruit
- rice bowl with lean protein
- protein shake + banana

4. Grocery Staples List
Simple grocery staples for staying on plan.

5. Real-Life Food Guidance
Suggestions for workdays, fast food swaps, quick snacks, and practical choices.

Nutrition should feel:
- simple
- practical
- supportive
- realistic
- not obsessive

WEEKLY CHECK-IN REQUIREMENTS

There should be a simple weekly check-in feature.

This should happen once per week, ideally on the Sunday / reset day.

Purpose:
- help users reflect
- increase engagement
- support accountability
- gather basic progress data
- give a sense of momentum

Keep it simple.

Suggested weekly check-in fields:
- workouts completed this week
- energy this week (1 to 5)
- how consistent were you this week
- biggest win
- biggest struggle
- optional weight entry
- optional progress photo upload later, if easy

V1 can start simple with just form inputs and saved check-in history.

COMPLETION FLOW REQUIREMENTS

Challenge completion is a major moment.

When the user completes the final challenge day, show a dedicated completion state.

Completion page/screen should include:
- congratulations message
- challenge completed summary
- encouragement
- next step offer / upsell

Example content:
Congratulations, you completed the 21-Day Challenge.
You showed up, stayed consistent, and finished strong.

Then show next step CTA like:
- Keep My Momentum Going
- Continue to the Next Level
- Join the Membership

The upsell should feel natural:
"You’ve already built momentum. Don’t lose it now."

USER STATES NEEDED

The system should account for these user states:

1. New user
- just joined
- has not started Day 1

2. In progress user
- completed some days
- next day should be highlighted

3. Returning user after gap
- missed time
- should see supportive restart messaging

4. Completed user
- finished all 21 days
- sees congratulations and upsell state
